Per Carrier Values

FORTUS™ is designed to be a comparitive rating engine, evaluating a risk for multiple insurance companies. Ideally, this design dictates the need for only one set of input data to be used by each insurance company's underwriting rules. For example, the <DwellingDate> is not going to change from one insurance company to another. In practice however, there are situations where information specific to the insurance company being evaluated is required. To accomplish this, Value Overrides are used. To specify a Value Override, add an attribute to the element. The name of the attribute must be "vor" (accronym for Value OverRide) plus the CSIO Company Code of the insurance company.

Example

Assume we're quoting ECON and DOC. To indicate a different liability amount for each use the following:

<Dwelling> ... <Liab vorECON="2000000" vorDOC="1000000">-1</Liab> </Dwelling>
