Input XML

Structures

The following tables outline the basic structure of the Input XML:

FortusTransaction
Attribute Name Type Length Range/Format Required Description
Type STRING   Table 1 Y Type of Fortus transaction
Tag Name Type Length Range/Format Required Description
Carriers Parent Tag     Y Holds 1 or more <Carrier> child tags, one for each carrier to be quoted.
Province STRING 2   Y Province codes (2 characters)
EffDate DATE   YYYYMMDD   Effective date of the quote.  If none is supplied or a value of 0 (zero) is used, then the current system date will be used.
Renewal STRING 3 YES/NO   YES = Policy Renewal, NO = New Business. Default: NO
CodeNames Parent Tag     Y Holds one or more codenames to dictate internal ratebook file associations.  A 'codename' is a Compu-Quote convention for identifying quote/dwelling/lookup type.
RegistryOverrides Parent Tag     Y Holds settings that take precedence over machine level configurations.
top

Carriers
Tag Name Type Length Range/Format Required Description
Carrier STRING 4   Y Compu-Quote Company Code of carrier to be quoted. This value must be completely in uppercase letters.
top

RegistryOverrides
Tag Name Range/Format Description
Diagnostics '1' or '0' A value of '1' indicates that diagnostic information should be included in the quote results. This can be very useful when trying to diagnose system/carrier problems.
HabMultipleForms '1' or '0' A value of '1' indicates that multiple forms for the requested form type should be quoted and returned if appropriate. A value of '0' indicates that the default form for the given form type should be assumed, thus returning only 1 set of quote results.
PrettyPrint '1' or '0' A value of '1' indicates that results XML should be formatted in a more easily human-readable fashion. This means that line-breaks and indentation will be used. However, doing this increases the size of the XML, so this is disabled by default.
StrictMap '1' or '0' A value of '1' indicates that any XML node that is not part of the map file does not below and a warning should be issued. This feature is rarely used.
top

CodeNames
Tag Name Type Length Range/Format Required Description
Auto Parent tag     Y Parent tag for Auto Quote Information
Hab* Parent tag   Table 2 Y Parent tag for Hab Quote Information
Property Parent tag     Y Parent tag for Hab 2 Quote Information

Examples

Auto Example

<FortusTransaction Type="Automobile"> <Carriers> <Carrier>AXA</Carrier> <Carrier>ROY</Carrier> </Carriers> <Province>ON</Province> <Renewal>NO</Renewal> <CodeNames> <Auto> <QuoteInfo> ... </QuoteInfo> </Auto> </CodeNames> </FortusTransaction>
top

Hab Example

<FortusTransaction Type="Property"> <Carriers> <Carrier>AXA</Carrier> <Carrier>ROY</Carrier> </Carriers> <Province>ON</Province> <EffDate>20021213</EffDate> <Renewal>NO</Renewal> <CodeNames> <Homeowners> <QuoteInfo> ... </QuoteInfo> </Homeowners> ... <Watercraft> <QuoteInfo> ... </QuoteInfo> </Watercraft> </CodeNames> </FortusTransaction>

Hab2 Example

Output XML

In the Output XML, each carrier's results are listed under the tag containing that carrier's code.

Examples

Auto Example

<FortusTransaction Type="Automobile"> <Carriers> <ROY> <CodeNames> <Auto> <QuoteInfo> ... </QuoteInfo> </Auto> </CodeNames> </ROY> <WEST> <CodeNames> <Auto> <QuoteInfo> ... </QuoteInfo> </Auto> </CodeNames> </WEST> </Carriers> ... </FortusTransaction>
top

Hab Example

<FortusTransaction Type="Property"> <Carriers> <GA> <CodeNames> <Homeowners> <QuoteInfo> ... </QuoteInfo> </Homeowners> <WaterCraft> <QuoteInfo> ... </QuoteInfo> </WaterCraft> </CodeNames> </GA> <ROY> <CodeNames> <Homeowners> <QuoteInfo> ... </QuoteInfo> </Homeowners> <WaterCraft> <QuoteInfo> ... </QuoteInfo> </WaterCraft> </CodeNames> </ROY> </Carriers> ... </FortusTransaction>
top

Hab2 Example

<FortusTransaction Type="Property2"> <Carriers> <GA> <CodeNames> <Property> <QuoteInfo> <Dwelling> ... </Dwelling> <Watercraft> ... </Watercraft> ... </QuoteInfo> </Property> </CodeNames> </GA> <ROY> <CodeNames> <Property> <QuoteInfo> <Dwelling> ... </Dwelling> <Watercraft> ... </Watercraft> ... </QuoteInfo> </Property> </CodeNames> </ROY> </Carriers> ... </FortusTransaction>
