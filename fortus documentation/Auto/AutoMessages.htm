Messages

Errors and Warnings relate to the 'process of quoting', whereas messages relate to the actual quote itself. Messages can be good, bad, or neutral. Some messages might simply be a means of relaying additional carrier specific information about the quote. Other messages might be intended on alerting the user to problems with the quote, and thus indicating unreliable quote results. Messages have been lumped together ambiguously like this to allow for flexibility. Each user implementing the FORTUS™ rating engine can then make their own decision on the severity and reaction to each individual message.

Auto Messages

Traditionally, common messages have been shared between all carriers. If a carrier-specific description was necessary then either message code 0 or 999 would have been used. Message functionality has now been expanded to allow for improved messaging as well as improved interpretation of those messages and rating situations. Table 1 describes messages in terms of what range they fall within. Table 2 provides the list of common messages used by all carriers.

Messages found in this range indicate a situation that cannot be confirmed because the necessary information is not collected. However, this does not hinder the quote results given. For example, these messages could indicate:

  • Discounts/Surcharges whose eligibility cannot be fully confirmed due to missing information not captured through FORTUS™.

600 - 699 Carrier informational messages.
Should not hinder quoting.

Messages found in this range represent carrier specific information. These messages often appear with no text. This is because these messages can be used by certain products (Rating Services, etc) as a mechanism for triggering desired post-quote functionality, rather than simply relaying message text to the end-user. Messages in this range without text can safely be ignored.

700 - 750 Validation Failures
Do not bind messages. Should not quote.

Messages found in this range represent a failure to validate certain necessary conditions. Their existence marks the quote results as not accurate or not complete.

Examples:

  • Refer to company for VRGs prior to binding.
  • The principal operator should be the highest rated operator
  • Does not qualify for special program.

This range is for future use. However, this range of message can still be programmed for and interpreted as do not bind messages. Quote results should not be used.

A message with this code can be generated for certain coverages that are not supported by the carrier. If all coverages requested on the quote are considered essential, then the quote results should not be used because not all coverages were quoted. Otherwise, the quote results can be used, with the caveat that not all requested coverages were quoted.
