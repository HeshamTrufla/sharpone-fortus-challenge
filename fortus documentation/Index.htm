FORTUS™

The Applied FORTUS™ Series 1 rating engine uses XML and SOAP 2 technology to provide a simple interface ideally suited for web rating systems.

Transactions

FORTUS™ can provide accurate quoting for insurers in real-time. Comparative rating is achieved in part with a common data structure. Data structures and their contents are defined outside the rating engine in a schema file called a MAP file. This MAP file is proprietary to the FORTUS™ engine.

FortusTransaction is the root node of all XML transactions with FORTUS™. While the Map files describe the data model used by the internal rating engines, some processing needs to occur at a higher level to ensure that the transaction is directed to the proper internal rating engine. This upper layer is thus not described in the Map Definition sections, but is instead described in the FortusTransaction section. The Type attribute of the FortusTransaction node indicates the type of transaction being requested, and the data model to be used.

  • <FortusTransaction Type="Automobile" ...> - Vehicle quoting with the Auto data model
  • <FortusTransaction Type="Property" ...> - Residential quoting with the Hab data model
  • <FortusTransaction Type="Property2" ...> - Residential quoting with the Hab 2 data model
