FORTUS™ XML Output

The output string returned by the FORTUS™ rating engine will always be well-formed XML. If errors and/or warnings occur during the quote process, FORTUS™ will capture and report them. See below for the different types of errors and warnings.

For errors, warnings, and messages that relate to rate calculation and/or rule underwriting, see the Messages section under each particular line of business: Auto, Hab, and Travel.

Parser Warnings - Generated by FORTUS™

The XML input is invalid in some way. Typically, the input XML has violated the map file definition. These warnings will generally only appear if Strict Map File Adherence is turned on.

Example

This example demonstrates what happens when a STRING tag contains a value that exceeds the listed maximum length.

<FortusTransaction Type="Automobile"> ... <ParserWarnings> <Warning> <Node>ID</Node> <NodeValue>ThisValueIsLongerThan20Characters</NodeValue> <ParentNode>QuoteInfo</ParentNode> <Message>Text value too big. Automatically truncated to fit.</Message> </Warning> </ParserWarnings> </FortusTransaction>

Parser Warnings and Errors - Generated by XML Parser (Xerces)

The XML input is invalid in some way. Ideally, these should be eliminated by the time you release. Warnings and errors generated by Xerces come in three flavors: warnings, errors and fatal errors. FORTUS™ places these items in the XML output and includes the same information about each. The only difference in appearance between the three is the parent tag: DOMWarning, DOMError, or DOMFatalError.

Example

This example demonstrates what happens when <SomeTag> is included in the input XML without a matching closing tag. The input XML is no longer well-formed because we have left out the closing tag, </SomeTag>.

<FortusTransaction Type="Automobile"> ... <ParsingErrors> <DOMFatalError> <Line>48</Line> <Column>15</Column> <Message>Expected end of tag 'SomeTag'</Message> </DOMFatalError> </ParsingErrors> </FortusTransaction>
top

Errors - Generated by FORTUS™

The existence of an XML Error structure indicates that a problem, crititcal enough to disqualify the quote results, has occurred. The <ErrorCode> and <Table> given can be used to additional information about the error in the Error Tables found below.

Example

In the following example, we use the <Table> value of FORTUS and the <ErrorCode> value of 41 to determine that FORTUS™ was not able to load the map file. Assuming FORTUS™ is being run on a Windows® system, then FortusConfig.exe should be run and the path to the map file (i.e. the Data Path) should be double checked.

<FortusTransaction Type="Automobile"> ... <Error> <ErrorCode>41</ErrorCode> <Table>FORTUS</Table> <Description>CQXLoaderLoad(HANDLE, "C:\CQFortus\data\invaliddir\auto\auto.map") returned -2. Unable to load map file.</Description> </Error> </FortusTransaction>
